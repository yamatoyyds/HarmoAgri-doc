# 开发计划

| 文档版本 | V1.1 |
|---|---|
| 创建日期 | 2026-03-15 |
| 更新日期 | 2026-03-16 |
| 文档作者 | OpenCode |

## 1. 文档目的

本文档用于把版本范围基线、开发阶段、输出物和风险收缩策略绑定起来，确保当前版本优先保障 MVP 与主演示路径。

## 2. 当前版本开发原则

- 先 P0，后 P1，再 P2
- 先主路径，后增强项
- 先闭环，后美化
- 先可交付，后扩展

## 3. 阶段划分

### 阶段 1：需求与方案冻结

- 完成 PRD 收紧
- 完成接口、页面、测试和追踪矩阵第一轮修订
- 冻结 MVP 范围和主演示路径

输出物：

- PRD
- 需求追踪矩阵
- 接口设计
- 页面详细规格
- 测试案例

进入条件：

- 赛题目标明确
- 文档整改范围已确认

退出条件：

- P0 需求定义完整
- MVP/ENHANCED/OPTIONAL 边界冻结

### 阶段 2：基础闭环实现

- 建立应用工程
- 打通首页数据展示
- 打通灌溉和补光控制闭环

输出物：

- 页面基础框架
- mock 数据链路
- 控制命令闭环

进入条件：

- 阶段 1 文档已冻结主要版本

退出条件：

- 首页看数可演示
- 手动灌溉和手动补光可完成提交与反馈

### 阶段 3：MVP 完整化

- 完成阈值告警
- 完成扩展设备接入
- 补齐页面失败和离线状态

输出物：

- 可演示 MVP 功能集
- 异常场景处理结果

进入条件：

- 基础闭环已打通

退出条件：

- P0 用例达到可执行状态
- MVP 功能具备回归基础

### 阶段 4：增强能力实现

- 完成自动补光
- 完成联动规则
- 补充历史记录等增强项

输出物：

- 增强功能验证结果
- 规则执行留痕能力

进入条件：

- 主路径稳定
- MVP 风险可控

退出条件：

- P1 功能具备演示条件或明确后置结论

### 阶段 5：测试与交付

- 执行功能、异常、联调和主演示链路测试
- 输出测试报告
- 完成答辩和演示材料

输出物：

- 测试报告
- PPT 和演示脚本
- 交付说明材料

进入条件：

- MVP 已完成
- 主路径基本稳定

退出条件：

- P0 用例全部通过
- TC-19 主演示链路一次通过

## 4. 周计划建议

### 第 1 周

- 完成文档第一轮和第二段修补
- 明确网关协议或 mock 协议
- 冻结 MVP 范围

### 第 2 周

- 建立应用工程框架
- 实现首页、设备页基础框架
- 接入实时数据 mock

### 第 3 周

- 实现灌溉和光照控制
- 打通设备控制接口
- 落实提交中、成功、失败状态

### 第 4 周

- 实现告警、扩展设备和离线态处理
- 开始 P0 回归测试

### 第 5 周

- 视风险决定是否补自动补光和联动规则
- 执行完整回归、联调和演示测试
- 输出测试报告和答辩材料

## 5. 责任分工建议

| 角色 | 主要职责 | 当前优先事项 |
|---|---|---|
| 产品/文档 | 需求、设计、交付材料 | 版本边界冻结、文档一致性 |
| 应用开发 | 页面、状态管理、接口联调 | 首页、灌溉、补光主路径 |
| 网关/设备联调 | 设备协议、控制链路、推送链路 | mock/真实链路双保障 |
| 测试 | 测试用例、缺陷跟踪、测试报告 | P0 用例与 TC-19 主链路 |

## 6. 风险触发后的替代路线

- 若硬件未就绪：先走 mock 网关，保住首页看数和控制闭环
- 若扩展设备不可用：采用模拟风扇设备保留扩展展示
- 若 P1 进度不足：后置联动规则和历史记录，优先保住 MVP

## 7. 完成标准

- 文档与实现保持一致
- P0 主功能闭环可演示
- 主演示路径一次通过
- 关键测试通过并可形成交付结论
