# 里程碑与任务拆解

|---|---|

## 1. 文档目的

本文档用于把版本范围基线、开发阶段和交付判定进一步拆解为里程碑和任务清单，保证“主路径优先、MVP 优先、增强项后置”的推进节奏。

## 2. 里程碑定义

- M1：需求与设计冻结
- M2：主路径基础闭环完成
- M3：MVP 功能完成
- M4：增强能力和扩展设备完成
- M5：测试和交付材料完成

## 3. 里程碑完成判定

| 里程碑 | 判定标准 |
|---|---|
| M1 | PRD、需求追踪矩阵、接口设计、页面详细规格、测试案例完成第一轮闭环 |
| M2 | 首页看数、手动灌溉、手动补光至少 3 条主路径打通 |
| M3 | 首页、网关状态、设备列表、灌溉、补光、基础告警、扩展设备可演示 |
| M4 | 自动补光、联动规则、历史记录等增强能力按风险情况完成或明确后置 |
| M5 | P0 用例全部通过，TC-19 一次通过，测试报告、PPT、演示脚本和交付文档完成 |

## 4. 任务拆解

### 4.1 文档类任务

- 收紧 PRD 与版本范围
- 完成需求追踪矩阵
- 收紧接口文档和字段字典
- 收紧页面规格和测试文档
- 完成项目计划、风险和里程碑文档联动

完成标准：

- 文档之间口径一致
- FR/NFR 可追踪
- 主路径和验收标准明确

### 4.2 开发类任务

- 首页总览展示
- 网关状态展示
- 设备列表与详情
- 手动灌溉控制
- 手动补光控制
- 告警展示
- 扩展设备接入

增强任务：

- 自动补光
- 联动规则
- 历史记录

### 4.3 联调类任务

- `/gateway/status` 联调
- `/telemetry/realtime` 联调
- `/devices` 联调
- `/devices/{id}/command` 联调
- WebSocket 推送联调

### 4.4 测试与交付类任务

- P0 用例执行
- 异常场景测试
- TC-19 主演示链路验证
- 测试报告输出
- PPT 和演示脚本输出

## 5. 推荐任务顺序

1. 冻结需求、接口和页面规格
2. 建立页面骨架和 mock 数据链路
3. 打通首页、灌溉和补光主路径
4. 完成基础告警与扩展设备接入
5. 视风险决定是否进入自动补光和联动规则
6. 执行测试和回归
7. 输出交付材料

## 6. 任务依赖说明

| 任务 | 依赖 |
|---|---|
| 页面骨架 | PRD、页面详细规格 |
| 接口联调 | 接口设计、字段字典、网关 mock/真实接口 |
| 控制闭环 | 设备在线、网关控制接口可用 |
| 告警展示 | 数据上报链路稳定、阈值规则可配置 |
| 扩展设备接入 | 统一设备抽象已稳定 |
| 测试执行 | MVP 主要功能已完成 |

## 7. 风险触发后的任务调整

- 若硬件未就绪：先执行 mock 路线相关任务
- 若 P0 主路径未稳定：暂停 P1 增强任务
- 若接口变化频繁：优先冻结契约并暂停非主路径功能实现

## 8. 当前执行建议

- 当前优先盯 M2 和 M3
- 在 M3 完成前，不应让 P2 需求进入主排期
- 每个里程碑结束后，应同步更新开发计划、风险管理和测试状态
