# 任务分工建议

## 1. 目标

为项目实施提供清晰的角色职责划分，避免多人协作过程中出现重复建设或责任空白。

## 2. 推荐角色分工

### 2.1 产品/文档负责人

职责：

- 维护 PRD
- 维护需求追踪矩阵
- 输出交付材料和答辩材料
- 协调需求变更

### 2.2 应用开发负责人

职责：

- 页面架构与状态管理
- 首页、设备页、灌溉页、光照页、告警页、设置页开发
- 接口接入和 UI 联调

### 2.3 网关/设备联调负责人

职责：

- 统一网关接口
- 设备协议适配
- 实时推送和控制链路保障
- 扩展设备接入验证

### 2.4 测试负责人

职责：

- 维护测试方案和测试案例
- 跟踪缺陷
- 输出测试报告

## 3. 推荐 RACI 表

| 任务 | 产品/文档 | 应用开发 | 网关/设备联调 | 测试 |
|---|---|---|---|---|
| PRD 编写 | R | C | C | C |
| 页面设计 | A | R | C | C |
| 接口定义 | C | C | R | C |
| 网关联调 | C | C | R | C |
| 功能开发 | C | R | C | C |
| 扩展设备接入 | C | C | R | C |
| 测试执行 | C | C | C | R |
| 测试报告 | C | C | C | R |
| PPT/答辩材料 | R | C | C | C |

说明：

- `R`：直接负责执行
- `A`：对结果负责
- `C`：参与协作

## 4. 协作建议

- 每周固定同步一次需求、接口和风险。
- 接口一旦冻结，非必要不轻易变更。
- 文档更新应同步通知应用开发和测试负责人。
