## 1. 文档目的

本文档用于将页面信息架构进一步细化为可直接指导 UI 开发、接口联调和测试编写的页面规格说明。

本文档重点补充以下内容：

- 字段展示规则
- 页面状态与状态切换
- 关键交互和禁止条件
- 成功、失败、离线、空态和部分成功态的表现方式

## 2. 通用页面规范

### 2.1 通用状态定义

| 状态 | 含义 | 页面要求 |
|---|---|---|
| loading | 首次加载 | 显示骨架屏或明确加载提示 |
| ready | 数据正常可用 | 展示完整主内容 |
| partial_error | 部分数据缺失 | 仅缺失区域降级，其他区域保持可用 |
| offline | 网关离线 | 显示全局异常提示，控制入口禁用 |
| empty | 当前无数据 | 显示引导文案 |
| submitting | 控制或保存提交中 | 禁止重复提交 |
| success | 操作成功 | 给出明确成功反馈 |
| failed | 操作失败 | 给出失败原因或错误码 |

### 2.2 通用交互规则

- 所有控制类操作在 `submitting` 状态下禁止重复点击
- 设备离线时，控制类按钮必须禁用，且展示“设备离线，不可控制”提示
- 网关离线时，页面仍可查看缓存或静态内容，但不可发起控制请求
- 保存类操作必须有成功或失败反馈，不能无感知结束
- 列表为空时应显示空态说明，而不是白屏

### 2.3 通用反馈规范

- 成功反馈：短提示 + 关键状态变更
- 失败反馈：短提示 + 失败原因或错误码
- 风险提示：对离线、不可达、规则冲突等情况给出明确文案
- 局部刷新：局部区域更新不应导致整页闪烁

## 3. 首页总览页规格

### 3.1 页面目标

- 在 3 秒内向用户展示环境是否正常、设备是否在线、是否存在异常告警

### 3.2 页面区块

| 区块 | 内容 | 数据来源 | 交互 |
|---|---|---|---|
| 顶部状态栏 | 网关状态、更新时间 | `GET /gateway/status` | 点击进入设置页 |
| 环境指标卡片区 | 温度、湿度、光照、二氧化碳、土壤湿度 | `GET /telemetry/realtime`、`telemetry.update` | 点击进入对应功能页 |
| 设备摘要区 | 在线设备数、离线设备数 | `GET /devices` | 点击进入设备页 |
| 告警摘要区 | 告警数量、最高等级告警 | `alert.created`、`GET /history/events?type=ALERT` | 点击进入告警页 |
| 快捷操作区 | 灌溉、补光、规则、设置 | 本地静态入口 | 点击跳转 |

### 3.3 字段定义

| 显示字段 | 说明 | 正常展示 | 异常展示 |
|---|---|---|---|
| 温度 | 当前温度值 | `24.5°C` | 缺失时显示 `--` |
| 湿度 | 当前空气湿度 | `61.2%` | 缺失时显示 `--` |
| 光照 | 当前光照强度 | `430 lux` | 低于阈值时高亮 |
| 二氧化碳 | 当前浓度 | `780 ppm` | 超阈值时高亮 |
| 土壤湿度 | 当前土壤湿度 | `38.1%` | 低于阈值时高亮 |
| 网关状态 | 在线/离线 | 在线标签 | 离线红色提示 |

### 3.4 页面状态切换

- 首次进入：`loading -> ready` 或 `loading -> partial_error` 或 `loading -> offline`
- 网关断开：`ready -> offline`
- 单项指标缺失：`ready -> partial_error`

### 3.5 关键交互

- 点击灌溉卡片进入灌溉页
- 点击补光卡片进入光照页
- 点击告警摘要进入告警页
- 点击网关状态进入设置页

### 3.6 失败和降级规则

- `/gateway/status` 失败时显示全局离线提示
- `/telemetry/realtime` 部分指标缺失时，仅对应卡片显示 `--`
- `/devices` 失败时设备摘要显示 `--`，但不阻塞环境卡片显示

## 4. 设备页规格

### 4.1 页面目标

- 统一展示所有设备，并支持查看设备详情和执行基础控制

### 4.2 页面区块

| 区块 | 内容 | 数据来源 | 交互 |
|---|---|---|---|
| 筛选区 | 按设备类型、在线状态筛选 | 本地状态 | 切换筛选条件 |
| 设备列表区 | 设备卡片列表 | `GET /devices` | 点击卡片打开详情 |
| 设备详情区 | 设备属性、最近状态、可执行动作 | `GET /devices`、`GET /history/events?type=COMMAND` | 执行控制 |

### 4.3 设备卡片字段

| 字段 | 说明 | 规则 |
|---|---|---|
| 设备名称 | 用户可识别名称 | 必显 |
| 设备类型 | PUMP/LED/FAN 等 | 必显 |
| 在线状态 | 在线/离线 | 离线时显著标记 |
| 当前状态 | ON/OFF/LEVEL 等 | 有则展示 |
| 所属区域 | 可选字段 | 无则隐藏 |

### 4.4 详情页动作

- 开启设备
- 关闭设备
- 调整亮度或档位（如适用）
- 查看最近控制记录

### 4.5 控制规则

- 离线设备：控制按钮禁用
- 在线设备：点击控制后进入 `submitting`
- 3 秒内未收到结果：显示“处理中，请稍候”
- 收到 `device.status.changed` 后更新最终状态

## 5. 灌溉页规格

### 5.1 页面目标

- 支持查看土壤湿度、查看灌溉建议和执行灌溉动作

### 5.2 页面区块

| 区块 | 内容 | 数据来源 | 交互 |
|---|---|---|---|
| 土壤湿度卡片 | 当前值、阈值状态 | `GET /telemetry/realtime` | 无 |
| 灌溉建议区 | 建议是否灌溉、建议说明 | 本地逻辑或规则结果 | 无 |
| 手动控制区 | 开启/停止、灌溉时长 | 用户输入 | 点击执行 |
| 自动配置区 | 自动灌溉开关、阈值 | `GET/POST /rules/automation` | 配置保存 |
| 记录区 | 最近灌溉日志 | `GET /history/events?type=COMMAND` | 查看历史 |

### 5.3 表单字段

| 字段 | 类型 | 必填 | 说明 | 校验规则 |
|---|---|---|---|---|
| autoEnabled | boolean | 否 | 是否开启自动灌溉 | 默认 `false` |
| soilThreshold | number | 是 | 土壤湿度阈值 | 0-100 |
| durationSec | number | 是 | 手动灌溉时长 | 1-600 秒 |

### 5.4 关键交互规则

- 点击“开始灌溉”前，若设备离线则直接提示，不发请求
- 点击“开始灌溉”后，按钮进入 `submitting`，防止重复点击
- 单次手动灌溉执行期间，自动灌溉规则不应覆盖当前操作
- 控制成功时更新记录区最新一条记录

### 5.5 页面状态

- `ready`
- `submitting`
- `success`
- `failed`
- `offline`

## 6. 光照页规格

### 6.1 页面目标

- 支持查看光照强度、执行手动补光和配置自动补光规则

### 6.2 页面区块

| 区块 | 内容 | 数据来源 | 交互 |
|---|---|---|---|
| 光照卡片 | 当前光照值、状态说明 | `GET /telemetry/realtime` | 无 |
| 手动控制区 | LED 开关、亮度等级 | 用户操作 | 点击执行 |
| 自动规则区 | 阈值、目标亮度 | `GET/POST /rules/automation` | 配置保存 |
| 记录区 | 最近补光记录 | `GET /history/events?type=COMMAND` | 查看历史 |

### 6.3 控制字段

| 字段 | 类型 | 必填 | 说明 | 校验规则 |
|---|---|---|---|---|
| lightThreshold | number | 是 | 光照阈值 | >= 0 |
| targetLevel | number | 是 | 目标亮度等级 | 1-5 |
| manualLevel | number | 否 | 手动亮度等级 | 1-5 |

### 6.4 交互规则

- 手动模式与自动模式需在 UI 上明确区分
- 自动模式开启时，页面展示当前阈值和目标亮度
- 手动执行补光时，若设备离线则禁止提交
- `SET_LEVEL` 成功后同步更新当前亮度展示

## 7. 告警页规格

### 7.1 页面目标

- 提供告警查看、筛选、详情追踪和状态管理能力

### 7.2 页面区块

| 区块 | 内容 | 数据来源 | 交互 |
|---|---|---|---|
| 筛选区 | 告警等级、状态、时间范围 | 本地筛选 | 设置条件 |
| 列表区 | 告警列表 | `alert.created`、`GET /history/events?type=ALERT` | 点击详情 |
| 详情区 | 告警内容、触发值、关联设备、处理状态 | `alert_event` | 确认/关闭 |

### 7.3 列表字段

| 字段 | 说明 | 是否必显 |
|---|---|---|
| 时间 | 告警触发时间 | 是 |
| 等级 | WARNING / CRITICAL | 是 |
| 类型 | 温度高、湿度低等 | 是 |
| 内容 | 告警描述 | 是 |
| 状态 | NEW / ACK / RESOLVED | 是 |

### 7.4 详情规则

- 详情至少展示等级、内容、触发值、状态、时间、关联设备
- `NEW` 状态可执行确认操作
- 已 `RESOLVED` 的告警只读展示，不提供确认按钮

## 8. 设置页规格

### 8.1 页面目标

- 管理网关连接、规则配置和扩展设备配置

### 8.2 子页面建议

- 网关配置页
- 阈值规则编辑页
- 联动规则编辑页
- 扩展设备配置页

### 8.3 网关配置字段

| 字段 | 类型 | 必填 | 说明 | 校验规则 |
|---|---|---|---|---|
| host | string | 是 | 网关地址 | 非空 |
| port | number | 是 | 网关端口 | 1-65535 |
| protocol | string | 是 | HTTP/HTTPS | 枚举校验 |
| wsPath | string | 否 | WebSocket 路径 | 可为空 |

### 8.4 保存规则

- 表单校验失败时禁止保存
- 保存成功时展示“保存成功”反馈
- 保存失败时展示失败原因或错误码

## 9. 增强项说明

### 9.1 历史记录展示

- V1 中历史记录作为增强项存在，不阻塞 MVP 交付
- 若进入实现，页面至少支持按 `ALERT`、`COMMAND`、`RULE` 类型查看记录
- 无记录时展示“当前暂无历史记录”空态

### 9.2 多设备区域化展示

- V1 中多设备区域化展示作为可选增强项，不进入当前版本交付约束
- 若进入实现，首页或设备页可按 `location` 字段做分组展示
- 区域化展示不应改变当前 MVP 页面主结构

## 10. 空态、错误态与离线态规范

### 10.1 空态

- 无设备：显示“当前暂无接入设备，请先完成设备接入”
- 无告警：显示“当前无异常告警”
- 无记录：显示“当前暂无历史记录”

### 10.2 错误态

- 数据请求失败：展示“数据加载失败，请重试”
- 控制失败：展示“控制失败，请检查设备状态或网络连接”
- 规则保存失败：展示“规则保存失败，请检查配置是否完整”

### 10.3 离线态

- 网关离线：展示全局离线条，控制入口禁用
- 设备离线：仅影响对应设备控制，不影响其他设备展示
