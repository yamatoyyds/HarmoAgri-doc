# 状态机与时序专项

## 1. 文档目的

统一展示系统中关键状态机和关键交互时序，便于设计评审、联调和答辩说明。

## 2. 网关连接状态机

```mermaid
stateDiagram-v2
    [*] --> INIT
    INIT --> CONNECTING
    CONNECTING --> CONNECTED
    CONNECTING --> FAILED
    CONNECTED --> DISCONNECTED
    FAILED --> CONNECTING
    DISCONNECTED --> CONNECTING
```

状态说明：

- `INIT`：应用刚启动，尚未尝试连接
- `CONNECTING`：正在连接网关
- `CONNECTED`：连接成功
- `FAILED`：连接失败
- `DISCONNECTED`：连接断开

## 3. 控制命令状态机

```mermaid
stateDiagram-v2
    [*] --> IDLE
    IDLE --> SUBMITTING
    SUBMITTING --> ACCEPTED
    SUBMITTING --> FAILED
    ACCEPTED --> SUCCESS
    ACCEPTED --> FAILED
    SUCCESS --> IDLE
    FAILED --> IDLE
```

状态说明：

- `IDLE`：未发起控制
- `SUBMITTING`：命令发送中
- `ACCEPTED`：网关已接收命令
- `SUCCESS`：设备状态已确认变更
- `FAILED`：命令失败或超时

## 5. 自动规则执行时序

```mermaid
sequenceDiagram
    participant Telemetry as 数据流
    participant Rule as RuleService
    participant Alert as AlertService
    participant Control as DeviceService
    participant GW as 网关

    Telemetry->>Rule: 上报实时指标
    Rule->>Rule: 匹配规则条件
    Rule->>Alert: 生成告警（如需要）
    Rule->>Control: 触发动作执行
    Control->>GW: 发送控制命令
    GW-->>Control: 返回执行结果
    Control-->>Rule: 回传结果
```

## 6. 首页加载时序

```mermaid
sequenceDiagram
    participant User as 用户
    participant Page as 首页
    participant GW as GatewayService
    participant DB as Repository/RDB

    User->>Page: 打开应用
    Page->>GW: 查询网关状态
    GW-->>Page: 返回状态
    Page->>GW: 拉取实时数据
    GW-->>Page: 返回实时数据
    Page->>DB: 读取最近告警/缓存
    DB-->>Page: 返回本地数据
    Page-->>User: 完成首页渲染
```

## 7. 手动灌溉控制时序

```mermaid
sequenceDiagram
    participant User as 用户
    participant Irrigation as 灌溉页
    participant Service as IrrigationService
    participant GW as 网关

    User->>Irrigation: 点击开启灌溉
    Irrigation->>Service: requestIrrigation()
    Service->>GW: POST /devices/{id}/command
    GW-->>Service: accepted
    GW-->>Service: 最终状态更新
    Service-->>Irrigation: 更新结果
```
