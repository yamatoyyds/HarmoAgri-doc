## 1. 测试目标

本文档用于将 PRD 中的功能需求、非功能需求和主演示路径转化为可执行、可判定的测试用例。

## 2. 优先级定义

- P0：必须通过，直接影响比赛演示闭环或交付判定
- P1：应通过，影响系统完整度和答辩说服力
- P2：增强验证项，不应阻塞当前版本交付

## 3. 功能测试案例

### TC-01 首页总览展示

- 优先级：P0
- 对应需求：FR-01、FR-02、FR-03、NFR-P-01
- 前置条件：应用正常启动，网关可访问，至少存在 1 个在线设备
- 测试步骤：
  1. 打开应用首页。
  2. 记录首页首屏完成展示的时间。
  3. 观察环境数据区域、网关状态区域和设备摘要区域。
- 预期结果：
  - 3 秒内完成首屏展示。
  - 首页展示温度、湿度、光照、二氧化碳、土壤湿度至少 5 项指标。
  - 首页展示网关在线状态和在线设备摘要。

### TC-02 手动灌溉控制

- 优先级：P0
- 对应需求：FR-04、FR-05、NFR-P-02
- 前置条件：水泵或水阀在线，灌溉页可正常打开
- 测试步骤：
  1. 进入灌溉页。
  2. 输入合法灌溉时长，例如 30 秒。
  3. 点击“开始灌溉”。
  4. 观察按钮状态和页面反馈。
- 预期结果：
  - 点击后立即进入 `submitting` 状态。
  - 3 秒内展示成功、失败或处理中状态。
  - 成功时设备状态变化可见，失败时有明确失败原因或错误码。

### TC-03 手动补光控制

- 优先级：P0
- 对应需求：FR-06、NFR-P-02
- 前置条件：LED 在线
- 测试步骤：
  1. 进入光照页。
  2. 调整亮度或切换开关。
  3. 观察页面反馈和状态更新。
- 预期结果：
  - 控制提交后进入 `submitting` 状态。
  - 3 秒内得到明确结果反馈。
  - 成功时 LED 状态或亮度变化可见。

### TC-04 告警触发

- 优先级：P1
- 对应需求：FR-08、FR-09
- 前置条件：已配置阈值规则
- 测试步骤：
  1. 构造超阈值数据。
  2. 刷新或等待推送。
  3. 观察首页和告警页。
- 预期结果：
  - 5 秒内生成告警记录或页面提示。
  - 首页和告警页均能看到异常提示。
  - 告警详情包含等级、内容、触发值、状态和时间。

### TC-05 自动联动触发

- 优先级：P1
- 对应需求：FR-10、FR-11
- 前置条件：已配置联动规则，目标设备在线
- 测试步骤：
  1. 构造规则触发条件。
  2. 观察设备动作和系统记录。
  3. 查看规则执行结果。
- 预期结果：
  - 系统自动执行目标动作。
  - 存在规则执行记录。
  - 规则失败时有明确失败状态。

### TC-06 扩展设备接入

- 优先级：P1
- 对应需求：FR-12、NFR-D-03
- 前置条件：风扇设备已注册
- 测试步骤：
  1. 进入设备页。
  2. 查看风扇设备状态。
  3. 执行开关控制。
- 预期结果：
  - 风扇设备可见且类型显示正确。
  - 风扇设备在线状态可见。
  - 控制成功后状态更新可见。

## 4. 异常测试案例

### TC-07 网关离线提示

- 优先级：P0
- 对应需求：NFR-S-01、NFR-S-03、NFR-D-02
- 前置条件：应用正常启动后断开网关
- 测试步骤：
  1. 进入首页。
  2. 断开网关服务。
  3. 观察首页、设备页和设置页。
- 预期结果：
  - 页面出现全局离线提示。
  - 控制动作不可执行或明确提示风险。
  - 应用不崩溃，用户可进入设置页执行重试。

### TC-08 设备离线控制失败

- 优先级：P0
- 对应需求：NFR-S-02、FR-04、FR-06
- 前置条件：目标设备标记为离线
- 测试步骤：
  1. 进入设备页或控制页。
  2. 尝试控制离线设备。
- 预期结果：
  - 页面阻止执行或返回失败提示。
  - 不发送无效控制请求，或返回 `1002`。
  - 其他在线设备不受影响。

### TC-09 数据缺失展示

- 优先级：P1
- 对应需求：NFR-S-02
- 前置条件：某类指标缺失
- 测试步骤：
  1. 模拟缺失光照数据。
  2. 观察首页和光照页。
- 预期结果：
  - 页面显示缺失态，例如 `--`。
  - 其他卡片和页面正常展示。

## 5. 联调测试案例

### TC-10 `/telemetry/realtime` 接口联调

- 优先级：P0
- 对应需求：FR-01、FR-02、FR-03、NFR-P-03
- 测试步骤：
  1. 触发首页加载。
  2. 检查接口返回字段和 `timestamp`。
  3. 核对页面展示值。
- 预期结果：
  - 接口返回成功结构符合约定。
  - `timestamp` 存在。
  - 页面与接口值一致。

### TC-11 `/devices/{id}/command` 控制联调

- 优先级：P0
- 对应需求：FR-04、FR-05、FR-06
- 测试步骤：
  1. 发送控制指令。
  2. 检查网关是否返回 `accepted`。
  3. 观察状态推送和页面更新。
- 预期结果：
  - 请求体带有 `requestId`。
  - 接口初始响应表示命令已接收。
  - 最终状态由推送补齐并与页面一致。

### TC-12 `alert.created` 推送联调

- 优先级：P1
- 对应需求：FR-08、FR-09
- 测试步骤：
  1. 构造超阈值环境数据。
  2. 观察 WebSocket 推送和页面更新。
- 预期结果：
  - 收到 `alert.created` 推送。
  - 告警页和首页提示同步更新。

### TC-13 `/rules/automation` 规则联调

- 优先级：P1
- 对应需求：FR-07、FR-10、FR-11
- 测试步骤：
  1. 提交完整规则配置。
  2. 提交缺失条件或缺失动作的非法规则。
  3. 观察返回结果。
- 预期结果：
  - 合法规则保存成功。
  - 非法规则返回参数错误或规则冲突错误。

## 6. 增强和边界测试案例

### TC-14 历史记录展示

- 优先级：P2
- 对应需求：FR-13
- 测试步骤：
  1. 打开历史记录区域。
  2. 查询 `COMMAND` 和 `ALERT` 类型数据。
- 预期结果：
  - 可展示历史列表。
  - 无数据时显示空态。

### TC-15 多设备区域化展示

- 优先级：P2
- 对应需求：FR-14
- 测试步骤：
  1. 准备多设备多区域数据。
  2. 打开首页或设备页。
- 预期结果：
  - 页面可按区域展示设备。

### TC-16 首页性能与刷新冒烟

- 优先级：P1
- 对应需求：NFR-P-01、NFR-P-03
- 测试步骤：
  1. 多次进入首页。
  2. 观察首屏展示时长和实时刷新表现。
- 预期结果：
  - 首屏 3 秒内可见。
  - 刷新不导致整页明显闪烁。

### TC-17 网关重连恢复

- 优先级：P1
- 对应需求：NFR-S-03
- 测试步骤：
  1. 断开网关。
  2. 恢复网关连接。
  3. 观察页面状态恢复情况。
- 预期结果：
  - 页面可从 `offline` 恢复到 `ready`。

### TC-18 错误响应与信息暴露检查

- 优先级：P1
- 对应需求：NFR-SEC-02、NFR-SEC-03
- 测试步骤：
  1. 构造非法请求。
  2. 观察错误响应结构和页面展示。
- 预期结果：
  - 错误响应结构统一。
  - 页面不直接暴露底层协议细节。

### TC-19 主演示链路一次通过验证

- 优先级：P0
- 对应需求：NFR-D-01
- 测试步骤：
  1. 按“首页看数 -> 手动灌溉 -> 手动补光/告警展示”的顺序执行完整演示。
  2. 记录是否需要重试或中断。
- 预期结果：
  - 主演示链路可连续执行一次通过。

### TC-20 文档追踪与扩展设计检查

- 优先级：P1
- 对应需求：NFR-M-01、NFR-M-02、NFR-M-03
- 测试步骤：
  1. 检查设备扩展、规则扩展和文档追踪关系是否在设计文档中闭环。
  2. 核对需求追踪矩阵、接口设计和详细设计是否一致。
- 预期结果：
  - 设备扩展和规则扩展有明确设计落点。
  - FR/NFR 与设计、测试保持一致映射。

### TC-21 兼容性与范围边界检查

- 优先级：P1
- 对应需求：NFR-C-01、NFR-C-02、NFR-C-03
- 测试步骤：
  1. 检查工程和设计文档是否明确 OpenHarmony 与 Stage 模型约束。
  2. 检查 PRD 是否明确单场景优先适配和区域化展示的可选属性。
- 预期结果：
  - 文档中明确 OpenHarmony 和 Stage 模型约束。
  - 区域化展示被定义为可选增强项，不影响 MVP 交付。
