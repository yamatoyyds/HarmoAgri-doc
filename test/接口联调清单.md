# 接口联调清单

## 1. 目的

明确前端与网关联调的接口范围、验证点和验收标准，减少联调遗漏。

## 2. 联调范围

| 接口 | 方法 | 用途 | 优先级 |
|---|---|---|---|
| `/gateway/status` | GET | 网关状态查询 | P0 |
| `/telemetry/realtime` | GET | 实时环境数据 | P0 |
| `/devices` | GET | 设备列表查询 | P0 |
| `/devices/{id}/command` | POST | 设备控制 | P0 |
| `/rules/alert` | GET/POST | 告警规则查询与保存 | P1 |
| `/rules/automation` | GET/POST | 联动规则查询与保存 | P1 |
| `/history/events` | GET | 历史记录查询 | P2 |
| `telemetry.update` | WS | 实时数据推送 | P0 |
| `device.status.changed` | WS | 设备状态推送 | P0 |
| `alert.created` | WS | 告警推送 | P1 |
| `rule.executed` | WS | 规则执行推送 | P1 |

## 3. 联调检查项

### 3.1 `/gateway/status`

- [ ] 返回字段完整
- [ ] 在线状态正确
- [ ] 心跳时间格式一致

### 3.2 `/telemetry/realtime`

- [ ] 至少返回 5 类环境指标中的有效数据
- [ ] 时间戳格式正确
- [ ] 缺失字段处理方式已约定

### 3.3 `/devices`

- [ ] 设备 ID 唯一
- [ ] 设备类型枚举统一
- [ ] 在线状态字段统一

### 3.4 `/devices/{id}/command`

- [ ] 支持 `requestId`
- [ ] accepted 返回语义明确
- [ ] 最终状态由推送或轮询补齐

### 3.5 WebSocket 推送

- [ ] topic 枚举统一
- [ ] data 字段结构稳定
- [ ] 异常断连可重连
