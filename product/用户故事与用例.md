## 1. 文档目的

本文档用于补充 PRD 中的用户角色、关键故事和核心用例，使需求更贴近真实使用场景，并为页面规格、接口设计和测试案例提供行为依据。

## 2. 角色清单

### 2.1 真实用户角色

| 角色 | 说明 | 核心目标 |
|---|---|---|
| 农户/操作员 | 日常使用者 | 快速看数、快速控制、快速确认结果 |
| 管理员 | 系统维护者 | 配置网关、配置规则、定位异常、维护设备 |

### 2.2 关键利益相关方

| 角色 | 说明 | 关注点 |
|---|---|---|
| 指导教师 | 项目指导者 | 方案合理性、文档完整度 |
| 评委/观察者 | 展示对象 | 演示稳定性、闭环完整性、扩展能力 |
| 项目负责人 | 交付责任人 | 版本边界、节奏和风险收缩 |

说明：评委/观察者不属于产品真实用户，不应直接驱动产品功能优先级，但应影响演示路径设计。

## 3. 用户故事列表

### 3.1 P0 用户故事

#### US-01 查看首页总览

作为农户，我希望在首页 3 秒内看到关键环境数据、网关状态和设备在线状态，以便快速判断当前种植环境是否正常。

#### US-02 手动灌溉

作为农户，我希望在土壤湿度过低时手动开启灌溉，并在短时间内看到执行结果，以便及时改善作物生长环境。

#### US-03 手动补光

作为农户，我希望在光照不足时手动开启 LED 补光或切换亮度，以便补足当前作物所需光照。

### 3.2 P1 用户故事

#### US-04 配置阈值告警

作为管理员，我希望为温度、湿度、光照和二氧化碳设置阈值，以便系统在异常时提醒我。

#### US-05 配置联动规则

作为管理员，我希望配置“当条件满足时自动执行控制”的规则，以便减少重复人工操作。

#### US-06 接入扩展设备

作为项目负责人，我希望系统能新增风扇等设备，以便证明系统具备扩展能力。

### 3.3 P2 用户故事

#### US-07 查看历史记录

作为管理员，我希望查看环境变化、告警和操作记录，以便回顾系统运行状态。

说明：US-07 在 V1 中仅作为增强项，不进入 MVP 验收主路径。

## 5. 核心用例描述

### UC-01：查看环境总览

- 参与者：农户/管理员
- 优先级：P0
- 前置条件：APP 已启动，网关服务可访问
- 触发条件：用户进入首页
- 成功标准：用户在 3 秒内看到至少 5 项环境指标、网关状态和设备摘要
- 基本流程：
  1. 系统请求网关状态和实时环境数据。
  2. 系统展示环境指标卡片。
  3. 系统展示设备在线状态摘要。
  4. 系统展示告警摘要。
- 异常流程：
  1. 如网关不可达，展示全局离线提示。
  2. 如单个传感器无数据，显示缺失状态但不阻塞整体页面。
- 后置条件：用户完成当前环境判断。

### UC-02：执行灌溉控制

- 参与者：农户/管理员
- 优先级：P0
- 前置条件：水泵或水阀在线，灌溉页可正常打开
- 触发条件：用户点击灌溉控制按钮
- 成功标准：3 秒内出现成功、失败或处理中状态
- 基本流程：
  1. 用户进入灌溉页并输入时长。
  2. 系统校验设备在线状态和表单合法性。
  3. 系统向网关发送控制命令。
  4. 页面进入 `submitting` 状态。
  5. 系统接收最终结果并更新状态。
- 异常流程：
  1. 设备离线，提示不可执行。
  2. 命令失败，显示失败原因或错误码。
  3. 超时未完成时，先显示处理中，再等待最终推送。
- 后置条件：设备状态和操作记录被更新。

### UC-03：执行手动补光

- 参与者：农户/管理员
- 优先级：P0
- 前置条件：LED 在线
- 触发条件：用户点击补光开关或切换亮度
- 成功标准：3 秒内出现明确反馈
- 基本流程：
  1. 用户进入光照页。
  2. 用户执行开关或亮度调整。
  3. 系统发送控制命令。
  4. 页面显示处理中并等待最终状态。
- 异常流程：
  1. 设备离线，禁止提交。
  2. 参数不合法，阻止提交并提示。
- 后置条件：页面展示最终设备状态。

### UC-04：配置阈值告警

- 参与者：管理员
- 优先级：P1
- 前置条件：系统已接入环境数据
- 触发条件：管理员进入阈值规则页
- 成功标准：支持至少 3 类阈值规则配置，异常后 5 秒内可见提示
- 基本流程：
  1. 选择指标类型和阈值。
  2. 保存规则。
  3. 规则生效后，在异常场景触发告警。
- 异常流程：
  1. 缺少指标或阈值时禁止保存。
  2. 重复或冲突规则返回失败提示。
- 后置条件：系统形成可追踪告警记录。

### UC-05：配置联动规则

- 参与者：管理员
- 优先级：P1
- 前置条件：系统已接入环境数据和设备
- 触发条件：管理员进入规则配置页
- 成功标准：支持至少 2 条联动规则并形成执行记录
- 基本流程：
  1. 选择触发条件，例如“土壤湿度低于阈值”。
  2. 选择执行动作，例如“开启灌溉”。
  3. 保存规则。
  4. 条件满足后系统自动执行。
- 异常流程：
  1. 规则条件配置不完整，禁止保存。
  2. 关联设备缺失或离线，记录失败结果。
- 后置条件：系统形成自动化联动闭环。

### UC-06：接入扩展设备

- 参与者：管理员/项目负责人
- 优先级：P1
- 前置条件：扩展设备支持统一设备模型接入
- 触发条件：新增扩展设备配置
- 成功标准：设备可展示状态并完成基础控制
- 基本流程：
  1. 接入扩展设备。
  2. 系统在设备页展示该设备。
  3. 用户执行基础控制。
- 异常流程：
  1. 设备模型不完整，禁止接入。
  2. 控制失败，展示失败结果。
- 后置条件：系统完成扩展性证明。
