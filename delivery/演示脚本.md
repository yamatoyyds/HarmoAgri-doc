# 演示脚本

## 1. 演示目标

在有限时间内完整展示系统的“看得见、控得住、能反馈、可扩展”。

## 2. 演示流程

### Step 1 首页总览

- 打开 APP 首页
- 展示温度、湿度、光照、二氧化碳、土壤湿度
- 展示设备在线状态

说明词示例：

- 这是系统首页，总览当前温室环境和设备状态。
- 用户无需进入多层页面即可快速判断环境是否正常。

### Step 2 手动灌溉

- 进入灌溉页
- 触发水泵或水阀开启
- 展示设备状态变化与结果反馈

说明词示例：

- 当土壤湿度偏低时，用户可以直接进行手动灌溉。
- 系统会将命令发送到网关，并回显执行状态，形成控制闭环。

### Step 3 手动补光

- 进入光照页
- 调整 LED 开关或亮度
- 展示补光状态更新

说明词示例：

- 当前页面展示光照强度与补光设备状态。
- 用户既可以手动补光，也可以切换到自动模式。

### Step 4 告警展示

- 构造异常阈值数据
- 展示告警产生、列表更新和首页提醒

说明词示例：

- 当环境指标超出阈值后，系统会自动生成告警。
- 告警既会出现在告警页，也会在首页摘要中提示。

### Step 5 联动规则

- 展示预设自动规则
- 构造触发条件
- 展示系统自动执行动作

说明词示例：

- 这里展示的是系统自动化能力，不依赖人工持续值守。
- 当条件满足时，系统会自动触发设备动作并记录结果。

### Step 6 扩展设备

- 展示风扇设备接入
- 执行风扇控制
- 强调系统扩展能力

说明词示例：

- 除赛题要求设备外，我们进一步接入了风扇设备。
- 这说明系统具备统一设备抽象和后续扩展能力。

## 3. 演示说明词建议

- 强调 OpenHarmony 原生架构
- 强调控制闭环与状态反馈
- 强调规则联动与扩展能力

## 4. 演示前检查清单

- [ ] 网关在线
- [ ] 关键设备在线
- [ ] 首页数据正常刷新
- [ ] 灌溉控制可执行
- [ ] 补光控制可执行
- [ ] 告警规则已配置
- [ ] 联动规则已配置
- [ ] 扩展设备可展示

## 5. 演示兜底策略

- 若真实设备不稳定，切换到 mock 数据与录屏方案。
- 若推送链路不稳定，使用静态可复现步骤演示。
- 若网络不稳定，优先展示本地缓存和文档材料。
