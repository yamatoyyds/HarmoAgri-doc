# 项目详细方案

## 1. 总体方案

本项目采用“设备层 - 网关层 - 应用层 - 展示层”的四层设计模式：

- 设备层采集环境数据并执行控制动作
- 网关层统一设备协议并提供对外接口
- 应用层负责业务逻辑、规则判断和数据持久化
- 展示层负责页面展示、状态反馈和用户交互

## 2. 核心能力

- 环境数据实时展示
- 灌溉控制
- 光照控制
- 阈值告警
- 自动联动
- 扩展设备接入

## 3. 技术方案

- Stage 模型
- UIAbility
- ArkTS + ArkUI
- Preferences + RDB
- HTTP + WebSocket

## 4. 实施策略

### 4.1 建设原则

- 先闭环、后增强
- 先原生、后混合
- 先稳定、后美化
- 先文档、后实现

### 4.2 MVP 范围

- 首页展示实时环境数据
- 设备列表与在线状态
- 手动灌溉控制
- 手动补光控制
- 告警规则基础能力
- 1 类扩展设备接入

### 4.3 增强范围

- 自动联动规则
- 历史记录
- 演示可视化优化
- 更丰富的异常处理展示

## 5. 系统亮点

### 5.1 技术亮点

- 基于 OpenHarmony 原生技术栈实现
- 采用网关抽象降低设备协议耦合
- 采用分层架构提升可扩展性

### 5.2 业务亮点

- 监测、控制、告警、联动构成完整闭环
- 扩展设备接入可验证平台能力
- 适合小型温室、教学与比赛演示场景

### 5.3 交付亮点

- 文档体系完整
- 测试思路完整
- 演示路径清晰

## 6. 演示方案

### 演示顺序建议

1. 首页展示环境总览
2. 查看设备状态
3. 手动执行灌溉控制
4. 手动或自动执行补光控制
5. 展示告警触发与处理
6. 展示扩展设备控制

### 6.1 演示时间建议

| 环节 | 建议时长 |
|---|---|
| 项目背景和架构介绍 | 1 分钟 |
| 首页与环境监测 | 1 分钟 |
| 灌溉与补光控制 | 1.5 分钟 |
| 告警与联动规则 | 1 分钟 |
| 扩展设备与总结 | 0.5 分钟 |

### 6.2 演示风险规避

- 演示前准备 mock 数据和真实数据两套方案
- 所有控制环节预先验证一次
- 关键页面保留静态截图作为兜底材料

## 7. 交付内容
