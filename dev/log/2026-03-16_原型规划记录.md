## 8. ArkUI 兼容性修正记录

- 已发现当前 DevEco Preview/ArkTS 编译环境对高阶 Builder 结构兼容性较弱。
- 已移除 `buildSection(..., () => { ... })` 形式的回调式 Builder 传参。
- 已移除 `@Builder` 方法中的局部 `const` 声明，改为直接调用成员方法。
- 已移除内联对象类型声明，改为显式接口定义。
- 已把部分内联数组抽为成员常量，降低 ArkTS 严格模式下的推断失败风险。
- 已将局部 UI 结构中的条件渲染收紧为更保守的属性控制方式，减少 Preview 解析错误概率。

## 9. 基于官方文档的代码复核结论

- 已查阅 `doc/OpenHarmonyDoc/LOCAL_DEV_DOC_INDEX.md` 以及 ArkTS 迁移、ArkUI UI 开发、状态管理等官方文档。
- 当前原型实现需继续遵守官方 ArkTS 约束：对象字面量必须具备显式类型、不得使用对象字面量声明类型、数组字面量元素需可推断。
- 当前原型代码虽然已修正一部分 Builder 与类型问题，但仍应继续朝“更少高阶写法、更少隐式推断、更强显式建模”的方向收紧。
