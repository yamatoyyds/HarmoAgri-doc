# 表结构设计

| 文档版本 | V1.1 |
|---|---|
| 创建日期 | 2026-03-15 |
| 更新日期 | 2026-03-16 |
| 文档作者 | OpenCode |

## 1. 设计原则

- 表结构需与接口字段、数据字典和需求追踪矩阵保持一致
- MVP 相关表优先保证可用和可追踪
- 所有时间字段统一使用毫秒时间戳 `INTEGER`
- 枚举值需与 `doc/db/数据字典.md` 保持一致

## 2. 设备表 `device`

| 字段名 | 类型 | 说明 | 约束 |
|---|---|---|---|
| id | TEXT | 设备唯一标识 | PRIMARY KEY |
| name | TEXT | 设备名称 | NOT NULL |
| type | TEXT | 设备类型 | NOT NULL |
| category | TEXT | `SENSOR` / `ACTUATOR` | NOT NULL |
| online_status | INTEGER | 在线状态 0/1 | NOT NULL |
| current_status | TEXT | 当前状态，如 `ON` / `OFF` |  |
| level | INTEGER | 档位或亮度等级 |  |
| location | TEXT | 所属区域 |  |
| extra_json | TEXT | 扩展信息 |  |
| created_at | INTEGER | 创建时间 | NOT NULL |
| updated_at | INTEGER | 更新时间 | NOT NULL |

建议索引：

- `idx_device_type`
- `idx_device_online_status`

## 3. 传感器记录表 `sensor_reading`

| 字段名 | 类型 | 说明 | 约束 |
|---|---|---|---|
| id | INTEGER | 自增主键 | PRIMARY KEY AUTOINCREMENT |
| device_id | TEXT | 关联设备 ID | NOT NULL |
| metric_type | TEXT | 指标类型 | NOT NULL |
| value | REAL | 指标值 | NOT NULL |
| unit | TEXT | 单位 |  |
| collected_at | INTEGER | 采集时间 | NOT NULL |

建议索引：

- `idx_sensor_reading_device_time (device_id, collected_at)`
- `idx_sensor_reading_metric_time (metric_type, collected_at)`

## 4. 告警规则表 `alert_rule`

| 字段名 | 类型 | 说明 | 约束 |
|---|---|---|---|
| id | TEXT | 规则 ID | PRIMARY KEY |
| name | TEXT | 规则名称 | NOT NULL |
| metric_type | TEXT | 指标类型 | NOT NULL |
| operator | TEXT | 比较符 | NOT NULL |
| threshold_value | REAL | 阈值 | NOT NULL |
| level | TEXT | `WARNING` / `CRITICAL` | NOT NULL |
| enabled | INTEGER | 是否启用 0/1 | NOT NULL |
| created_at | INTEGER | 创建时间 | NOT NULL |
| updated_at | INTEGER | 更新时间 | NOT NULL |

约束建议：

- 同一指标、比较符、阈值、等级的重复规则应在业务层阻止

## 5. 告警事件表 `alert_event`

| 字段名 | 类型 | 说明 | 约束 |
|---|---|---|---|
| id | TEXT | 事件 ID | PRIMARY KEY |
| rule_id | TEXT | 规则 ID |  |
| device_id | TEXT | 设备 ID |  |
| metric_type | TEXT | 指标类型 | NOT NULL |
| level | TEXT | 告警等级 | NOT NULL |
| message | TEXT | 告警内容 | NOT NULL |
| status | TEXT | `NEW` / `ACK` / `RESOLVED` | NOT NULL |
| triggered_value | REAL | 触发值 |  |
| created_at | INTEGER | 触发时间 | NOT NULL |
| resolved_at | INTEGER | 解除时间 |  |

建议索引：

- `idx_alert_event_status_time (status, created_at)`
- `idx_alert_event_level_time (level, created_at)`

## 6. 联动规则表 `automation_rule`

| 字段名 | 类型 | 说明 | 约束 |
|---|---|---|---|
| id | TEXT | 规则 ID | PRIMARY KEY |
| name | TEXT | 规则名称 | NOT NULL |
| condition_json | TEXT | 条件定义 | NOT NULL |
| action_json | TEXT | 动作定义 | NOT NULL |
| enabled | INTEGER | 是否启用 0/1 | NOT NULL |
| created_at | INTEGER | 创建时间 | NOT NULL |
| updated_at | INTEGER | 更新时间 | NOT NULL |

约束建议：

- 保存前需校验 `condition_json` 和 `action_json` 的结构完整性

## 7. 控制记录表 `command_record`

| 字段名 | 类型 | 说明 | 约束 |
|---|---|---|---|
| id | TEXT | 记录 ID，可与 `requestId` 一致 | PRIMARY KEY |
| device_id | TEXT | 设备 ID | NOT NULL |
| command_type | TEXT | 命令类型 | NOT NULL |
| request_payload | TEXT | 请求内容 |  |
| result_status | TEXT | `ACCEPTED` / `SUCCESS` / `FAILED` | NOT NULL |
| result_message | TEXT | 执行结果说明 |  |
| source | TEXT | `MANUAL` / `AUTOMATION` | NOT NULL |
| created_at | INTEGER | 创建时间 | NOT NULL |
| updated_at | INTEGER | 更新时间 | NOT NULL |

建议索引：

- `idx_command_record_device_time (device_id, created_at)`
- `idx_command_record_status_time (result_status, created_at)`

## 8. 推荐外键关系

- `sensor_reading.device_id -> device.id`
- `alert_event.device_id -> device.id`
- `command_record.device_id -> device.id`
- `alert_event.rule_id -> alert_rule.id`

说明：若运行环境不强依赖数据库外键，可在业务层保证数据一致性，但关系定义仍需在设计层明确。

## 9. 数据保留建议

- `sensor_reading`：保留近 30 天原始记录，超期可聚合或归档
- `command_record`：保留比赛周期全部记录
- `alert_event`：保留比赛周期全部记录
- `automation_rule`、`alert_rule`：保留当前有效配置和变更后的最终版本
